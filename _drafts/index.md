# 草稿文档

本目录用于存放未完成的文档草稿和临时内容。

## 使用说明

- 存放正在编写但未完成的文档
- 临时收集的资料和想法
- 需要进一步整理的内容

## 工作流程

1. **创建草稿**：在相应子目录创建草稿文档
2. **标记状态**：使用YAML frontmatter中的`status: draft`
3. **定期整理**：将完成的草稿移至正式目录
4. **清理无用**：删除无价值的草稿内容

## 命名规范

- 使用描述性的文件名
- 在文件名中标注草稿状态（如：draft-xxx.md）
- 定期更新和维护
